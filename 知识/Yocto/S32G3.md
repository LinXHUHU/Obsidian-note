# 无密码获取sudo 权限

在Linux系统中，如果你想要无密码执行 `sudo apt-get <cmd>` 命令，你需要配置 `sudoers` 文件来允许特定用户或用户组无密码执行 `sudo` 命令。以下是如何安全地做到这一点的步骤：

### 方法 1：使用 `visudo` 命令

1. 打开终端。
2. 输入 `sudo visudo` 命令并按回车。这将使用默认的文本编辑器打开 `sudoers` 文件。
3. 在 `sudoers` 文件中找到以下行（可能被注释掉了）：

   ```
   %sudo   ALL=(ALL:ALL) ALL
   ```

4. 如果你想要为特定用户无密码执行 `sudo` 命令，取消注释上面的行，并在 `ALL` 之前添加该用户的用户名，如下所示：

   ```
   yourusername ALL=(ALL:ALL) NOPASSWD: ALL
   ```

   或者，如果你只想让该用户无密码执行 `apt-get` 命令，可以这样配置：

   ```
   yourusername ALL=(ALL) NOPASSWD: /usr/bin/apt-get
   ```

5. 保存并关闭文件。在 `visudo` 中，通常可以通过按 `Ctrl + X`，然后按 `Y`（表示是），最后按回车来保存更改。

### 方法 2：使用 `sudoedit` 命令

如果你不想使用 `visudo`，也可以使用 `sudoedit` 命令来编辑 `sudoers` 文件：

1. 输入 `sudo sudoedit` 命令并按回车。
2. 在打开的编辑器中，找到或添加上面提到的行。
3. 保存并退出编辑器。

### 注意事项：

- **安全性**：允许无密码 `sudo` 可以带来安全风险，因为它允许指定的用户绕过密码提示直接执行高权限命令。确保你完全信任该用户，并且了解可能的安全后果。
- **特定命令**：如果你只想允许无密码执行 `apt-get` 命令，确保在 `NOPASSWD` 行中指定 `/usr/bin/apt-get` 路径。这是 `apt-get` 命令的完整路径，确保 `sudo` 可以正确识别。
- **用户组**：你也可以为整个用户组配置无密码 `sudo`，而不是单个用户。只需将 `yourusername` 替换为 `%yourgroupname`。

请谨慎操作，并确保你了解这些更改的影响。
