
# 项目初始化

这是一个标准的 **pre-commit 配置文件** (`.pre-commit-config.yaml`)，用于在代码提交前自动执行代码质量检查工具。以下是对配置的逐项解析：

---

### **1. 文件作用**
- **pre-commit** 是一个 Git 钩子管理框架，在代码提交前（`git commit`）自动运行指定工具，确保代码符合规范。
- 该配置文件定义了要运行的检查工具（hooks）及其配置。

---

### **2. 配置解析**
#### **第一部分：基础代码检查**
```yaml
repos:
-   repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v3.2.0  # 使用版本 v3.2.0
    hooks:
    -   id: trailing-whitespace  # 删除行尾空格
    -   id: end-of-file-fixer    # 确保文件末尾有换行符
    -   id: check-yaml          # 检查 YAML 文件语法
    -   id: check-added-large-files  # 阻止提交大文件（默认 >500KB）
```
- **作用**：自动修复基础格式问题，避免低级错误。

#### **第二部分：Python 代码格式化**
```yaml
-   repo: https://github.com/pycqa/isort  # Python import 排序工具
    rev: 5.12.0
    hooks:
    -   id: isort
        stages: [push]  # 仅在 git push 时运行（非 commit 时）
```
- **isort**：自动规范化 Python 的 import 语句顺序（如标准库、第三方库、本地库分组）。

```yaml
-   repo: https://github.com/psf/black  # Python 代码格式化工具
    rev: 22.10.0
    hooks:
    -   id: black
        stages: [push]  # 仅在 git push 时运行
```
- **black**：强制统一 Python 代码风格（如缩进、引号等）。

---

### **3. 关键概念**
| 配置项       | 说明                                                                 |
|--------------|----------------------------------------------------------------------|
| **repo**     | 钩子工具所在的 Git 仓库地址                                          |
| **rev**      | 指定工具版本（推荐固定版本号，避免更新导致意外行为）                 |
| **hooks.id** | 要启用的具体钩子（工具提供的子功能）                                 |
| **stages**   | 触发阶段：默认是 `commit`，可设为 `push`/`manual` 等                 |

---

### **4. 使用流程**
1. **安装 pre-commit**：
   ```bash
   pip install pre-commit
   ```
2. **安装钩子**（将钩子写入 `.git/hooks`）：
   ```bash
   pre-commit install
   ```
3. **手动运行所有钩子**：
   ```bash
   pre-commit run --all-files
   ```
4. **提交代码时**：自动触发配置的钩子，若检查失败会阻止提交。

---

### **5. 自定义建议**
#### **(1) 添加 Rust 相关钩子**
若项目包含 Rust 代码，可添加：
```yaml
-   repo: https://github.com/doublify/pre-commit-rust
    rev: v1.0
    hooks:
    -   id: cargo-check
    -   id: fmt
    -   id: clippy
```

#### **(2) 调整检查阶段**
- 将耗时工具（如 `black`）设为 `stages: [push]`，避免每次提交都运行。
- 快速检查（如空格修复）保持默认 `commit` 阶段。

#### **(3) 跳过钩子**
临时跳过检查（需谨慎）：
```bash
git commit --no-verify
```

---

### **6. 配置文件示例扩展**
完整示例（含更多工具）：
```yaml
repos:
-   repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.4.0
    hooks:
    -   id: trailing-whitespace
    -   id: end-of-file-fixer
    -   id: check-yaml
    -   id: check-json
    -   id: check-added-large-files
        args: [--maxkb=1024]  # 自定义大文件阈值

-   repo: https://github.com/psf/black
    rev: 23.3.0
    hooks:
    -   id: black
        args: [--line-length=88]  # 设置行长度

-   repo: https://github.com/pycqa/flake8
    rev: 6.0.0
    hooks:
    -   id: flake8
        additional_dependencies: [flake8-bugbear==23.3.12]
```

---



通过 pre-commit，你可以将代码审查的许多低级检查自动化，让团队更专注于逻辑而非风格问题！
