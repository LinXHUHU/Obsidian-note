
## 🔁 1. 整体系统软件更新（Overall System Software Update）

### ✅ 是什么？
- 系统中的每个域（MCU Domain、Secure Domain、Main Domain）都包含自己的固件或软件。
- 软件更新需要协调所有相关域，确保更新过程安全、一致，并且在失败时能够回滚。

### 🧩 涉及哪些域？
- **Secure Domain**：负责验证新版本签名是否合法。
- **MCU Domain**：控制整个系统的重启和升级流程。
- **Main Domain**：运行完整的 OTA 升级逻辑，处理 QNX、Adaptive AUTOSAR、AOS 应用等的更新。

### ⚠️ 关键点：
- 更新必须是原子性的（要么全成功，要么全回滚）。
- 需要支持差分更新、断点续传、失败回滚机制。
- 安全性要求高，需使用加密传输 + 数字签名验证。

## 📜 2. 所有域的日志管理（All Domains Log Management）

### ✅ 是什么？
- 每个域都会产生日志（例如：启动日志、错误日志、诊断日志、调试信息等）。
- 需要统一收集、存储、分析这些日志，以便于调试、故障排查、合规审计等。

### 🧩 涉及哪些域？
- **MCU Domain**：记录底层硬件状态、电源管理事件等。
- **Secure Domain**：记录安全相关事件（如密钥访问、签名验证失败等）。
- **Main Domain**：记录应用层、中间件、自动驾驶算法等运行日志。

### ⚠️ 关键点：
- 日志格式需要标准化，便于集中处理。
- 支持远程上传日志到云端进行分析。
- 某些日志可能需要加密保存，尤其是来自 Secure Domain 的敏感日志。

## 🚨 3. 事件管理（Event Management）

### ✅ 是什么？
- 监控系统中发生的各类事件（如异常、故障、用户操作、传感器触发等），并根据优先级进行响应。
- 事件可以来自任意一个域，但处理逻辑可能需要跨域协作。

### 🧩 涉及哪些域？
- **MCU Domain**：上报电源切换、看门狗超时、CAN通信异常等。
- **Secure Domain**：上报安全事件（如非法访问、签名验证失败等）。
- **Main Domain**：上报应用层错误、传感器数据异常、路径规划失败等。

### ⚠️ 关键点：
- 需要有统一的事件 ID 和分类标准。
- 支持事件优先级分级（Critical / Warning / Info）。
- 可能触发自动降级策略、用户提示、OTA 上报等动作。

## 🛡️ 4. 功能安全（Functional Safety）

### ✅ 是什么？
- 指的是系统在面对软硬件故障时，仍能保证车辆不会因故障导致危险情况的能力。
- 符合 ISO 26262 标准，覆盖从设计、开发到运行的全过程。

### 🧩 涉及哪些域？
- **MCU Domain**：实现 ASIL-D 级别的监控与防护机制。
- **Secure Domain**：提供安全认证、密钥保护、可信执行环境。
- **Main Domain**：运行感知、决策、控制等模块，需满足 ASIL 要求。

### ⚠️ 关键点：
- 需要跨域的故障检测与恢复机制。
- 包括看门狗监控、内存保护、任务调度检查等。
- 支持 FMEA（失效模式分析）、FTA（故障树分析）等方法。

## 🔄 5. 域间通信（Inter Node Communication）

### ✅ 是什么？
- 不同域之间需要传递数据、调用服务、共享资源。
- 由于每个域可能运行不同的操作系统或执行环境，因此通信方式需要特别设计。

### 🧩 涉及哪些域？
- **MCU Domain ↔ Main Domain**：通过 IPC、SPI、Mailbox 等方式进行通信。
- **Secure Domain ↔ Main Domain**：通过 TrustZone 提供的 SMC（Secure Monitor Call）机制调用安全服务。
- **MCU Domain ↔ Secure Domain**：通常通过 A Core 中介进行间接通信。

### ⚠️ 关键点：
- 需要定义统一的数据接口和服务调用规范。
- 对于敏感通信，需加密或签名验证。
- 通信延迟需控制在可接受范围内。

---

## 🔍 6. 诊断管理（Diagnostics Management）

### ✅ 是什么？
- 实现 UDS（统一诊断服务）等协议，用于读取故障码（DTC）、清除故障、读写参数、刷写ECU等。
- 支持 OBDII、DoIP 等标准接口。

### 🧩 涉及哪些域？
- **MCU Domain**：处理底层 ECU 的诊断请求。
- **Secure Domain**：对诊断命令进行安全验证，防止非法刷写。
- **Main Domain**：处理高级别诊断请求，如自动驾驶模块的状态查询。

### ⚠️ 关键点：
- 诊断命令需要身份认证和权限控制。
- 支持跨域诊断调用（如主域调用 MCU Domain 的诊断接口）。
- 支持远程诊断（Remote Diagnostics）功能。
