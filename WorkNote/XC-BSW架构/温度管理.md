## 📌 一、模块基本信息

| 项目 | 内容 |
|------|------|
| 模块名称 | Thermal Management（热管理模块） |
| 模块ID | [1019] |
| 功能安全等级 | ASIL B |
| 核心职责 | 温度监控、硬件保护、风扇控制、状态通知 |

---

## 🔧 二、模块功能详解

### 1. **温度采集（Temperature Monitoring）**
- **功能描述**：
  - 周期性地从 **温度传感器（如SoC内部传感器、外部NTC等）** 获取温度数据。
  - 支持多个温度源（如CPU、GPU、电源模块、外壳等）。
- **实现方式**：
  - 调用底层驱动接口（如 `Exe_TempSensorDriver`）读取温度。
  - 设置采样周期（如每 100ms 或 500ms 一次）。

---

### 2. **硬件保护（Hardware Protection）**
- **功能描述**：
  - 当检测到 **温度超过安全阈值（Critical Thermal Condition）** 时，采取保护措施。
- **可能的措施**：
  - **降频**（Throttling）：降低CPU/GPU频率以减少发热量。
  - **关闭非关键域**（Domain Shutdown）：请求 LCM（Life Cycle Manager）关闭主域或部分模块。
  - **强制关机**：若温度极高且无法控制，执行系统关机以防止硬件损坏。

---

### 3. **风扇控制（Fan Control）**
- **功能描述**：
  - 根据当前温度动态调节风扇转速。
- **实现方式**：
  - 输出 PWM 信号控制风扇转速。
  - 支持多级控制策略（如线性控制、查表控制）。
- **示例策略**：
  | 温度范围 | 风扇转速 |
  |----------|----------|
  | < 60°C   | 0%（关闭）|
  | 60~80°C  | 50%       |
  | 80~90°C  | 80%       |
  | > 90°C   | 100%      |

---

### 4. **温度信息提供（Temperature Info Provision）**
- **功能描述**：
  - 向其他模块（如诊断模块、FCT模块）提供实时温度信息。
- **接口方式**：
  - 提供统一的接口（如 `Thermal_GetCurrentTemp()`）供其他组件调用。
  - 支持事件通知机制（如回调函数或事件广播）。

---

### 5. **热状态通知（Thermal State Notification）**
- **功能描述**：
  - 根据温度变化，向其他模块（如 Diagnosis、FCT）通知当前热状态。
- **热状态定义示例**：
  ```c
  typedef enum {
      THERMAL_STATE_NORMAL,      // 正常
      THERMAL_STATE_WARN,        // 警告
      THERMAL_STATE_CRITICAL,    // 严重
      THERMAL_STATE_SHUTDOWN     // 关机
  } ThermalStateType;
  ```
- **通知方式**：
  - 调用 Diagnosis 模块的接口上报状态（如 `Diag_ReportThermalState()`）。
  - 向 FCT 模块发送事件通知（如通过 IPC 或事件总线）。

---

## 🧱 四、与 LCM 的交互逻辑

### **请求主域关闭或恢复（Domain Shutdown / Recover）**
- **触发条件**：
  - 温度超过临界阈值（Critical Temp） → 请求 LCM 关闭主域。
  - 温度下降至恢复阈值 → 请求 LCM 恢复主域运行。
- **交互方式**：
  - 调用 LCM 的接口（如 `LCM_RequestDomainShutdown()` 或 `LCM_RequestDomainRecover()`）。
  - 传递参数：域ID、请求类型、优先级等。

---

## 🔐 五、ASIL B 安全需求与实现建议

| 安全目标 | 实现方式 |
|----------|-----------|
| **故障检测** | 周期性自检、传感器故障检测（如开路/短路） |
| **冗余机制** | 多个温度传感器互为备份，防止单点失效 |
| **安全状态** | 默认进入关闭模式，防止误操作 |
| **时序监控** | 看门狗机制，确保温度采集与控制任务按时执行 |
| **故障注入测试** | 支持模拟传感器失效、通信中断等场景 |
