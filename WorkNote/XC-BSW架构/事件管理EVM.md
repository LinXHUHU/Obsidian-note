## 📌 一、EvM 模块定义

**EvM（Event Management）** 是 **事件管理模块**，其核心作用是：

> **负责在系统中收集、分类、处理和上报来自各个监控器（Monitors）的故障事件（fault events）**。

EvM 是 AUTOSAR 架构中 **DEM（Diagnostic Event Manager）** 的扩展概念，适用于 **多核异构架构（如 A-Core + R-Core）** 和 **多域架构（如车身域、动力域、自动驾驶域）**。

---

## 🧱 二、EvM 模块的核心作用

### ✅ 1. **事件采集（Event Collection）**
- **来源**：从本地域的监控器（Monitors）收集故障事件（如传感器失效、通信失败、看门狗超时等）。
- **事件类型**：
  - 硬件故障事件（如电源异常）
  - 软件故障事件（如断言失败、看门狗触发）
  - 通信故障事件（如 CAN 错误帧、以太网丢包）

### ✅ 2. **事件分类与优先级处理**
- 根据事件的严重性（如 Warning、Error、Critical）进行分类。
- 设置优先级（Priority）以便在资源受限时优先处理关键事件。

### ✅ 3. **事件上报（Distribution of FIDs）**
- 将采集到的事件封装为 **FID（Fault ID）** 或 **Event ID**。
- 向上层模块（如 DEM/FIM）或远程域上报事件。
- 支持同步/异步上报机制。

### ✅ 4. **事件本地处理（可选）**
- 在本地进行初步处理，如：
  - 触发安全机制（如关闭域、进入降级模式）
  - 记录日志
  - 启动恢复流程

### ✅ 5. **跨域通信支持**
- 与 R-CORE 中的 **DEM/FIM** 进行通信，实现事件的集中处理。
- 支持 IPC、共享内存、RPMSG 等跨核通信机制。


| 模块 | 交互方式 | 说明 |
|------|-----------|------|
| **Monitors** | 本地调用 | EvM 从监控模块获取事件 |
| **DEM/FIM** | IPC / 共享内存 | 将事件上报至中心处理模块 |
| **Diag模块** | 接口调用 | 提供诊断访问接口 |
| **DTCManager** | 接口调用 | 用于记录故障码 |
| **LCM / FCT** | 事件通知 | 根据事件触发系统行为（如重启、降级） |

## 📦 四、EvM 的部署结构（J6E JETOUR 项目）

| Core/Domain | EvM 部署 | 说明 |
|-------------|----------|------|
| **R-CORE** | ✅ 是中心节点 | 部署 DEM/FIM，负责集中处理所有事件 |
| **A-CORE** | ✅ 部署 EvM 模块 | 采集本地事件并上报至 R-CORE |
| **其他域（如车身域、底盘域）** | ✅ 部署 EvM 模块 | 实现事件本地采集与上报 |

## 🔐 六、功能安全（FUSA）与 EvM 的关系

EvM 模块是 **FUSA 架构中的关键组件**，特别是在 ASIL 分级系统中：

| FUSA 要求 | EvM 的实现方式 |
|-----------|----------------|
| **故障检测** | 通过 Monitor 上报事件实现 |
| **故障记录** | DEM/FIM 记录 DTC |
| **故障响应** | EvM 通知 LCM/FCT 触发安全机制 |
| **跨核一致性** | 使用共享内存或 IPC 保证事件同步 |
| **故障注入测试** | 支持模拟事件注入用于测试 |

## 📈 七、EvM 的典型应用场景

| 场景 | 说明 |
|------|------|
| **传感器失效** | Monitor 检测到传感器通信失败，EvM 上报事件 |
| **看门狗超时** | 看门狗模块触发事件，EvM 上报并触发系统复位 |
| **通信失败** | CAN/Ethernet 通信异常，EvM 上报并记录 DTC |
| **OTA升级失败** | 升级模块上报失败事件，EvM 通知 DEM 记录并回滚 |
| **安全状态切换** | 根据事件触发系统进入安全状态（如驾驶降级） |
