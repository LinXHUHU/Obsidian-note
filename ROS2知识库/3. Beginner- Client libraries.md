

**Colcon** (Common Command Line Build Tool) 是 ROS 2 的默认构建系统。它是为了统一和简化不同构建系统（如 CMake, Python setuptools, Gradle 等）的操作而设计的。

- **ROS 1 时代**:
    - `catkin_make`: 早期的标准构建工具，简单但功能有限。
    - `catkin_make_isolated`: 用于隔离构建包，避免依赖干扰，但速度较慢。
    - `catkin_tools` (`catkin build`): 社区驱动的强大工具，支持并行构建和更好的错误处理，但未成为官方默认。
- **过渡期**:
    - `ament_tools`: ROS 2 早期使用的构建工具，专为 `ament` 构建系统设计。
- **ROS 2 现在**:
    - **`colcon`**: 集成了上述所有工具的优点。它不仅支持 ROS 的 `ament_cmake` 和 `ament_python`，还能构建普通的 CMake 项目、Python 包甚至 Rust/Cargo 项目。它是**通用、快速且可扩展**的。


### 🛠️ 关键概念：工作空间结构

在使用 `colcon` 之前，你需要了解标准的 ROS 2 工作空间目录结构：


```
my_ros2_ws/                # 工作空间根目录
├── src/                   # 【重要】源代码目录
│   ├── my_package_1/      # 包 1
│   └── my_package_2/      # 包 2
├── build/                 # 构建过程中的中间文件 (缓存、对象文件)
├── install/               # 【重要】安装目录 (最终的可执行文件、库、环境变量)
└── log/                   # 构建日志
```

- **`src/`**: 你必须手动创建并将你的代码包放在这里。`colcon` 会扫描这个目录。
- **`build/`**: `colcon build` 时自动生成，通常不需要手动干预。
- **`install/`**: `colcon build` 时自动生成。**这是最关键的部分**，构建完成后，你需要 `source install/setup.bash` 来让系统识别新生成的节点。




`--symlink-install` 是 ROS 2 开发者的**效率神器**。它弥补了 ROS 2 取消 `devel space` 后带来的迭代不便，特别适合主要使用 **Python** 或频繁调整 **配置/Launch 文件** 的开发场景。

**推荐习惯**：在个人开发机器上进行功能开发和调试时，始终使用 `colcon build --symlink-install`。只有在准备正式部署或打包时，才进行干净的标准构建。
