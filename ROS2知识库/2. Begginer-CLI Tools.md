# 环境配置

 **ROS 2 是一个基于环境变量的生态系统。**

- **Underlay** = 官方基础。
- **Overlay** = 你的定制开发。
- **Source** = 开启这个生态系统的钥匙。

**一定要养成习惯**：每开始一个新的开发会话，确保你已经 source 了正确的 setup 文件！

_注意：不要在同一个终端里混合 source 两个不同版本的 ROS，这会导致库冲突和奇怪的错误。_


### `ROS_LOCALHOST_ONLY` 极简指南

**作用**：  
将 ROS 2 通信限制在**本机**。开启后，你的节点**无法**与局域网内的其他电脑/机器人通信，也看不到它们的数据。

**为什么用**：

- **防干扰**：在教室、实验室等多人环境中，防止误收别人的话题（如 `/cmd_vel`）导致机器人乱跑。
- **更干净**：调试时只看本机数据，避免网络广播噪音。



### 1. 三大核心工具

| 工具 | 类型 | 作用 | 特点 |  
| :--- | :--- | :--- | : |  
| **Turtlesim** | **模拟器** | 轻量级仿真环境，用于演示 ROS 2 基本原理。 | 就像编程界的 "Hello World"。让你在不连接真实硬件的情况下，体验控制机器人的感觉（移动、画图等）。 |  
| **`ros2`** | **命令行工具 (CLI)** | 管理、检查和交互的核心工具。 | **全能型**。所有操作（启动节点、监听话题、设置参数等）都可以通过它完成。是底层最强大的接口。 |  
| **`rqt`** | **图形界面 (GUI)** | 可视化的操作工具。 | **友好型**。功能与 `ros2` 命令相同，但通过图表、按钮和窗口展示，更适合直观地监控数据流和调试。 |
